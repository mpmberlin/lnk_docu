## 1. Address Score

`POST` /check/crypto_address/score

Retrieves a normalized AML risk score and detailed risk metadata for a given blockchain address.
A higher score means greater AML risk; the API also classifies it into four severity bands:
Low: 0 – 30; 
Moderate: 31 – 70;
High: 71 – 90;
Severe: 91 – 100;

### Response fields
- **`risk_score`** (Type: Integer):  
  Normalized AML risk rating from 3 to 100; higher is more risky.
- **`hacking_event`** (Type: Saddamtring):  
  Name of any known breach or exploit linked to this address/transaction; empty if none.
- **`risk_score`** (Type: Integer):  
  Normalized AML risk rating from 3 to 100; higher is more risky.
- **`flags`** (Type: Integer):  
  Human-readable tags summarizing key risk drivers.
- **`risk_detail`** (Array[Object]):  
  - **`label`** (String): Risk category name (e.g. `"Scam Token"`, `"Phishing"`).  
  - **`type`** (String): Classification (`"malicious"`, `"suspected_malicious"`, `"high_risk"`).  
  - **`volume`** (Number): Amount transacted in this incident.  
  - **`address`** (String): Counterparty or related address.  
  - **`percent`** (Number): Fraction of total volume exposed by this incident (e.g. `0.453` = 45.3%).

## 2. Address actions
`POST` /check/crypto_address/action

Summarizes the count and share of inbound and outbound transactions by category

### Response fields

- **`received_txs`** (Type: Array[Object])  
  List of inbound transaction categories. Each object contains:
  - **`action`** (String): Category name (e.g. “Swap”, “DEX”, “Bridge”).  
  - **`count`** (Integer): Number of transactions in this category.  
  - **`proportion`** (Number): Percentage of total received transactions (e.g. `27.08` = 27.08%).

- **`spent_txs`** (Type: Array[Object])  
  List of outbound transaction categories. Each object contains the same fields as `received_txs`:
